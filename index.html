Vellum

Edit any page. Hand the changes to your agent.

Vellum is a drop-in overlay that turns every text node on a page into agent-ready markdown. Click. Edit. Copy. Paste.

npm i vellum

How you use it

  1. Drop vellum into your page — a <script> tag, a bookmarklet, or npm i vellum.
  2. The toolbar appears bottom-right.
  3. Click any text. It becomes editable inline.
  4. Type your edit. The count increments.
  5. Click Copy markdown in the toolbar.
  6. Paste into your AI coding agent: "apply these edits to the codebase."

How agents use it

Vellum's output gives agents enough context to find and replace exactly what you changed.

  • A CSS selector path (e.g. main > section > h1) for grepping markup.
  • The exact before and after text, preserved verbatim, for grepping copy.
  • An element name (h1 "Pricing", paragraph: "Lorem…") so the agent can disambiguate visually similar nodes.
  • The URL, pathname, and viewport the edit was made on.
  • A removed-element status when the DOM no longer contains the original target.

Best practices

  • Keep edits surgical — one fix per element.
  • Reset before switching tasks so the next agent prompt starts clean.
  • Use click-to-arm mode when reviewing read-heavy pages where you might fat-finger a heading.
  • Mark dynamic regions with data-vellum-ignore to keep them out of the edit set.
  • Trust the before text — agents grep for it. If it doesn't match the source, fix the markup, don't fight vellum.
